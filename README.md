# Лабораторна Робота №3:Тема: Моделі Вводу/Виводу: Практичне порівняння: написання echo-сервера з використанням блокуючого I/O, epoll та io_uring.
## Опис Завдання
Мета роботи:
Практична: Реалізувати простий TCP "echo-сервер" з використанням трьох фундаментально різних моделей I/O, детально розглянутих у лекції.
Архітектурна: Зрозуміти, як вибір моделі I/O (блокуюча, мультиплексування, асинхронна) кардинально змінює архітектуру та логіку серверного додатку.
Аналітична: На практиці відчути та проаналізувати переваги та недоліки кожного підходу з точки зору складності коду, управління станом та потенційної продуктивності.
Інструментальна: Набути навичок роботи з низькорівневими системними викликами Linux (epoll_create, epoll_ctl, epoll_wait) та сучасною бібліотекою liburing для io_uring.

## Реалізовані Моделі

### 1. Блокуючий I/O (Blocking I/O)
* **Директорія:** `blocking_server/`
* **Модель:** 1 процес на клієнта (`fork()`).
* **Опис:** Класичний підхід, де кожен клієнт обробляється окремим дочірнім процесом.

### 2. Мультиплексування I/O (Epoll)
* **Директорія:** `epoll_server/`
* **Модель:** Однопотоковий Event Loop.
* **Опис:** Використання неблокуючих сокетів та механізму `epoll` для ефективної обробки тисяч з'єднань в одному потоці.

### 3. Асинхронний I/O (io_uring)
* **Директорія:** `io_uring_server/`
* **Модель:** Сповіщення про завершення.
* **Опис:** Використання сучасного асинхронного інтерфейсу ядра Linux (`liburing`) для передачі операцій I/O безпосередньо в ядро.

## Інструкції для Збірки та Запуску

Увійдіть у відповідну директорію (`blocking_server/`, `epoll_server/` або `io_uring_server/`) та виконайте:

1.  **Компіляція:** `make`
2.  **Запуск:** `./server`
3.  **Тестування:** Підключіться через `nc 127.0.0.1 8080`
